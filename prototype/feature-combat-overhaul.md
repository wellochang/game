# 戰鬥機制改版 Feature Spec

版本：draft　日期：2026-05-15

---

## 背景與設計原則

現行戰鬥為純自動的 1D 追趕，雙方以最高速行駛，沒有玩家輸入。問題：

- 若玩家速度慢且武器射程短，雙方永遠不在射程內，只能撤退（僵局）
- 配裝未成型時可能遇到完全天克的敵人，全無應對手段，死亡只感覺不公平

**設計目標：** 死亡維持硬結局（廢土氛圍），但讓玩家的每一步都有選擇——死得其所，而不是死於資訊不足或無路可走。

---

## Feature 一：節點進入前的敵方情報

### 設計

玩家點選地圖節點前，顯示一條敵方局部情報：**隨機揭露武器或車體其中一個**。

```
示意：
[RAIDERS]  情報：裝備 Mortar
[WARLORD]  情報：Juggernaut 車架
```

### 規則

- 普通戰（fight）：顯示**射程（gap-px）**
- 精英戰（elite）：顯示**車體型號**（kind）
- Boss：兩者都顯示（屬於已知強敵）
- 資訊不對稱是設計意圖：知道射程不知道速度，知道車體不知道火力

### 設計意圖

情報讓玩家有憑據做決策，但不給全圖：
- 知道射程 280 → 警覺是中程敵人，但不知道速度與 HP
- 知道是 Juggernaut → 知道慢但耐打，但不知道近戰還是遠程
- 老玩家比新玩家更能解讀情報，遊戲知識有價值

## Feature 二：消耗 FUEL 跳過節點

### 設計

玩家在**進入節點前**可以選擇「繞路」，消耗 FUEL 跳過本次遭遇，不進入戰鬥、不取得獎勵。

### 規則

| 節點類型 | 可否跳過 | FUEL 費用 |
|----------|----------|-----------|
| fight    | 是       | 1         |
| elite    | 是       | 2         |
| scrap    | 不需要   | —         |
| event    | 是       | 1         |
| boss     | **否**   | 硬牆      |

- 跳過不得任何獎勵（廢料、HP 回復、事件效果）
- FUEL 耗盡後無法跳過，只能打或退出遊戲

### FUEL 補充管道

| 來源     | 補充量 |
|----------|--------|
| 廢料場新增選項「補充燃料」 | +3 FUEL，費用 2 廢料 |
| 隨機事件偶發 | 視事件而定 |

### 設計意圖

FUEL 終於作為策略資源接入機制。「沒油就只能賭」是廢土殘酷感的高峰：不是遊戲逼你死，是你自己把油燒完了。

## Feature 三：速度區塊系統（Zone System）

> 取代原本的連續速度值與檔位設計，是 Feature 四的完整重構版本。

### 核心概念

拿掉絕對速度值，改為 **Z1–Z5 五個離散速度區塊**。玩家和敵方各自維持當前區塊，gap 由雙方區塊的速度差決定。

### 區塊速度對照

| 區塊 | 速度係數 | 描述 |
|------|----------|------|
| Z1 | 0.4 | 龜速 |
| Z2 | 0.8 | 慢速 |
| Z3 | 1.2 | 中速（戰鬥起始） |
| Z4 | 1.8 | 快速 |
| Z5 | 2.6 | 全速衝刺 |

*數值為基準，實作時依手感調整。*

### Gap 計算

```
// gap 連續積分（每 dt 更新，非回合結算）
gap = clamp(gap - (playerSpd - enemySpd) × 60 × dt, 0, 800)
```

每 **12 秒**觸發一次玩家選區介面（`ROUND_DURATION = 12`）。

gap 縮小 → 玩家區塊速度高於敵方（玩家追上）
gap 增大 → 敵方區塊速度高於玩家（敵方拉開）

---

### 引擎決定加速度（可選區塊範圍）

每回合玩家可選的區塊 = `[current_zone - accel, current_zone + accel]`，夾在 Z1–Z5。

| 引擎 | 加速度 | 在 Z3 可選 | 特性 |
|------|--------|------------|------|
| Diesel | ±1 | Z2、Z3、Z4 | 穩定，換區慢 |
| Nitro  | ±2 | Z1、Z2、Z3、Z4、Z5 | 反應快，可跳兩格 |
| Hybrid | ±1 | Z2、Z3、Z4 | 同 Diesel，失控率低（見下） |

---

### 開場情境：由節點類型決定

| 節點 | 情境 | 玩家起始 | 敵方起始 | 敘事邏輯 |
|------|------|----------|----------|----------|
| fight | 玩家被追 | Z3 | Z2 | 流寇從後方殺來，速度略慢但貼近 |
| elite | 玩家追 | Z3 | Z4 | 軍閥在逃，玩家需要追上 |
| boss  | 玩家追 | Z3 | Z4 | 最終追擊，男爵全速撤退 |

> 此為模板。未來 roguelike 隨機性可在此基礎上做變體（如：某些 fight 觸發「玩家追」情境）。

**開場 gap 動態分析（12 秒 / 回合）：**

```
被追（fight）：玩家 Z3(1.2) vs 敵方 Z2(0.8)
  gap_delta = (1.2 - 0.8) × 60 × 12 = +288 px/回合（gap 縮小）
  → 從初始 gap 450 約 1.5 回合可觸發衝撞

追（elite/boss）：玩家 Z3(1.2) vs 敵方 Z4(1.8)
  gap_delta = (1.2 - 1.8) × 60 × 12 = -432 px/回合（gap 增大）
  → 玩家若維持 Z3，需在 2 回合內升至 Z4+ 才能追上
```

---

### 敵方 AI：區塊選擇邏輯

敵方每回合選擇區塊（±1），目標是讓 zone 差落在自身武器 `wZoneRange` 內：

```
diff = abs(playerZone - enemyZone)

if diff > wZoneRange:
    往玩家 zone 靠近 ±1     // 追近到射程內
else:
    維持當前 zone           // 已在射程內，不動
```

> 敵方目前沒有主動拉開距離的邏輯：若玩家闖入射程內，敵方不會後退。

**流派拉鋸範例：**

| 玩家流派 | 敵方類型 | 動態 |
|----------|----------|------|
| 遠程（Mortar Z3） | 近程衝撞（Z0） | 敵方持續追近，玩家退 zone 拉大差距，Mortar 一路打 |
| 近程（Flamer Z0） | 遠程砲台（Z2） | 玩家衝高 zone，敵方追到 Z2 射程後停，玩家需超越才能貼臉 |
| 中程（Autocannon Z2） | 中程對手（Z1） | 雙方 Z3-Z4 拉鋸，對打 |

---

### 車架穩定性機制

#### 大型車（Hauler）：高速失控

切換至高區塊時有機率失控，直接掉回 Z2：

| 目標區塊 | 失控機率 |
|----------|----------|
| Z4 | 20% |
| Z5 | 45% |

失控時：`actual_zone = Z2`（強制降速，視覺上底盤震動）

Hybrid 引擎：失控機率各 -10%（Z4→10%，Z5→35%）

#### 小型車（Scout）：受擊失速

每次受到傷害觸發判定：

```
20% 機率：current_zone = max(Z1, current_zone - 1)
```

代表輕車被打中後失去節奏，Scout 在近戰中有額外風險。

---

### 回合時序

```
[回合開始]
  → 戰鬥暫停
  → 顯示選區介面：可選區塊按鈕 + 當前 gap + 各武器射程狀態
  → 玩家選擇目標區塊
  → 敵方 AI 同步選擇目標區塊
[回合進行 12 秒]
  → 武器依 zoneDiff <= w.zoneRange 自動開火
  → 處理失控判定（Hauler）/ 受擊判定（Scout）
  → gap 連續更新（速度差 × 60 × dt）
[回合結束，重複]
```

### 流派對應

| 流派 | 慣用操作 | 戰術 |
|------|----------|------|
| 遠程（Mortar 560） | 維持低區塊 | gap 自然拉大，壓在敵方射程外 |
| 中程（Autocannon 400） | 跟住 Z3-Z4 | 對打，靠裝甲撐 |
| 近程（Flamer 160） | 衝 Z5 | 強行貼臉，Scout 需接受受擊風險 |
