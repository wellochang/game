# 戰鬥機制改版 Feature Spec

版本：draft　日期：2026-05-15

---

## 背景與設計原則

現行戰鬥為純自動的 1D 追趕，雙方以最高速行駛，沒有玩家輸入。問題：

- 若玩家速度慢且武器射程短，雙方永遠不在射程內，只能撤退（僵局）
- 配裝未成型時可能遇到完全天克的敵人，全無應對手段，死亡只感覺不公平

**設計目標：** 死亡維持硬結局（廢土氛圍），但讓玩家的每一步都有選擇——死得其所，而不是死於資訊不足或無路可走。

---

## Feature 一：節點進入前的敵方情報

### 設計

玩家點選地圖節點前，顯示一條敵方局部情報：**隨機揭露武器或車體其中一個**。

```
示意：
[RAIDERS]  情報：裝備 Mortar
[WARLORD]  情報：Juggernaut 車架
```

### 規則

- 普通戰（fight）：顯示**武器**
- 精英戰（elite）：顯示**車體**
- Boss：兩者都顯示（屬於已知強敵）
- 資訊不對稱是設計意圖：知道武器不知道速度，知道車體不知道火力

### 設計意圖

情報讓玩家有憑據做決策，但不給全圖：
- 知道有 Mortar → 警覺射程遠，但不知道能不能追上
- 知道是 Juggernaut → 知道慢但耐打，但不知道近戰還是遠程
- 老玩家比新玩家更能解讀情報，遊戲知識有價值

## Feature 二：消耗 FUEL 跳過節點

### 設計

玩家在**進入節點前**可以選擇「繞路」，消耗 FUEL 跳過本次遭遇，不進入戰鬥、不取得獎勵。

### 規則

| 節點類型 | 可否跳過 | FUEL 費用 |
|----------|----------|-----------|
| fight    | 是       | 1         |
| elite    | 是       | 2         |
| scrap    | 不需要   | —         |
| event    | 是       | 1         |
| boss     | **否**   | 硬牆      |

- 跳過不得任何獎勵（廢料、HP 回復、事件效果）
- FUEL 耗盡後無法跳過，只能打或退出遊戲

### FUEL 補充管道

| 來源     | 補充量 |
|----------|--------|
| 廢料場新增選項「補充燃料」 | +3 FUEL，費用 2 廢料 |
| 隨機事件偶發 | 視事件而定 |

### 設計意圖

FUEL 終於作為策略資源接入機制。「沒油就只能賭」是廢土殘酷感的高峰：不是遊戲逼你死，是你自己把油燒完了。

## Feature 三：敵方定位 AI

### 設計意圖

敵方根據自身武器射程，主動調整速度維持最佳射擊距離。這不是「讓著玩家」，而是敵方有自己的戰術定位意圖——近程敵方想貼近、遠程敵方想保持距離。玩家和敵方同時在爭奪 gap 的控制權。

### 核心公式

```
gap -= (player.spd - enemy.spd) × 60 × dt
```

gap 完全由速度差決定。雙方各自調速，形成拉鋸。

### 敵方 AI 邏輯

```
enemy.tgt = enemy.wRange × 0.7     // 維持在自己射程的 70% 處

if gap > enemy.tgt + 30:
    enemy.targetSpd = max(0.5, player.spd × 0.85)   // 減速，讓玩家靠近
elif gap < enemy.tgt - 30:
    enemy.targetSpd = enemy.maxSpd                    // 加速，拉開距離
else:
    enemy.targetSpd = enemy.spd                       // 維持

每幀平滑收斂：
enemy.spd += (enemy.targetSpd - enemy.spd) × min(1, dt × 2.5)
```

### 拉鋸場景

| 玩家意圖 | 敵方意圖 | gap 走向 |
|----------|----------|----------|
| 高速追近 | 短程（也想靠近） | 雙方趨近，gap 快速縮小 |
| 高速追近 | 長程（想拉遠） | 敵方加速逃，玩家追，緩慢拉鋸 |
| 低速後退 | 短程（想靠近） | 敵方跟進減速，gap 微縮或靜止 |
| 低速後退 | 長程（也想拉遠） | 雙方拉開，gap 快速增大 |

### 數字範例

玩家 `maxSpd=2.0`，敵方（短程）`maxSpd=2.8`，`wRange=180`，`enemy.tgt=126`：

```
初始 gap=450，玩家高速：
  player.spd = 2.0
  gap > 126 → enemy.targetSpd = 2.0 × 0.85 = 1.7
  gap 變化 = -(2.0 - 1.7) × 60 = -18 px/s
  → 約 18 秒後 gap 縮至 126

玩家低速：
  player.spd = 1.0
  gap > 126 → enemy.targetSpd = 1.0 × 0.85 = 0.85
  gap 變化 = -(1.0 - 0.85) × 60 = -9 px/s
  → gap 仍縮，但速度減半（短程敵方低速仍追得上）
```

### 不對稱性（設計意圖）

- **玩家 vs 近程敵方**：不管玩家選哪個檔，gap 都在縮小，差別只是快慢。想靠低速維持遠距，需要敵方 maxSpd < 玩家半速才可行
- **玩家 vs 遠程敵方**：敵方加速跑，玩家高速才追得上；若敵方 maxSpd > 玩家 maxSpd，玩家永遠追不上 → 這是速度劣勢的真實代價，靠情報系統提前判斷

## Feature 四：玩家檔位選擇（回合制速度控制）

### 設計

戰鬥中每隔固定秒數暫停，讓玩家選擇**下一段的行進檔位**，選完後繼續自動戰鬥。

### 玩家速度公式

```
高速檔：player.targetSpd = player.maxSpd
低速檔：player.targetSpd = player.maxSpd × 0.5

每幀平滑收斂：
player.spd += (player.targetSpd - player.spd) × min(1, dt × 3)
```

### 流派對應

| 流派 | 慣用檔位 | 戰術邏輯 |
|------|----------|----------|
| 遠程（Mortar 560） | 低速 | 壓在敵方射程外，一路砲轟 |
| 中程（Autocannon 400） | 高速 | 快速進場，靠裝甲對打 |
| 近程（Flamer 160） | 高速衝刺 | 強行貼臉，需速度優勢或敵方也想靠近 |

### 時序

- 每個回合長度：**3 秒**（約一個武器 CD 循環）
- 選擇方式：**暫停選擇**（戰鬥停下，玩家點選，然後繼續）
- 選擇畫面：兩個按鈕 + 當前 gap、武器射程狀態

### 待決定

- 2 檔還是 3 檔（中速 = 維持當前 gap 是否有意義）
- 高速是否消耗 FUEL
- 首回合預設高速，還是強制先選

## 實作優先序建議

| 優先 | Feature | 理由 |
|------|---------|------|
| 1    | 敵方追逐 AI | 解決僵局，影響最大，改動最小 |
| 2    | 節點情報顯示 | UI 層改動，直接提升決策品質 |
| 3    | FUEL 跳過節點 | 需要補充管道一起設計 |
| 4    | 檔位系統 | 最複雜，需要 UI 新增 |
