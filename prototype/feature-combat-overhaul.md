# 戰鬥機制改版 Feature Spec

版本：draft　日期：2026-05-15

---

## 背景與設計原則

現行戰鬥為純自動的 1D 追趕，雙方以最高速行駛，沒有玩家輸入。問題：

- 若玩家速度慢且武器射程短，雙方永遠不在射程內，只能撤退（僵局）
- 配裝未成型時可能遇到完全天克的敵人，全無應對手段，死亡只感覺不公平

**設計目標：** 死亡維持硬結局（廢土氛圍），但讓玩家的每一步都有選擇——死得其所，而不是死於資訊不足或無路可走。

---

## Feature 一：節點進入前的敵方情報

### 設計

玩家點選地圖節點前，顯示一條敵方局部情報：**隨機揭露武器或車體其中一個**。

```
示意：
[RAIDERS]  情報：裝備 Mortar
[WARLORD]  情報：Juggernaut 車架
```

### 規則

- 普通戰（fight）：顯示**武器**
- 精英戰（elite）：顯示**車體**
- Boss：兩者都顯示（屬於已知強敵）
- 資訊不對稱是設計意圖：知道武器不知道速度，知道車體不知道火力

### 設計意圖

情報讓玩家有憑據做決策，但不給全圖：
- 知道有 Mortar → 警覺射程遠，但不知道能不能追上
- 知道是 Juggernaut → 知道慢但耐打，但不知道近戰還是遠程
- 老玩家比新玩家更能解讀情報，遊戲知識有價值

## Feature 二：消耗 FUEL 跳過節點

### 設計

玩家在**進入節點前**可以選擇「繞路」，消耗 FUEL 跳過本次遭遇，不進入戰鬥、不取得獎勵。

### 規則

| 節點類型 | 可否跳過 | FUEL 費用 |
|----------|----------|-----------|
| fight    | 是       | 1         |
| elite    | 是       | 2         |
| scrap    | 不需要   | —         |
| event    | 是       | 1         |
| boss     | **否**   | 硬牆      |

- 跳過不得任何獎勵（廢料、HP 回復、事件效果）
- FUEL 耗盡後無法跳過，只能打或退出遊戲

### FUEL 補充管道

| 來源     | 補充量 |
|----------|--------|
| 廢料場新增選項「補充燃料」 | +3 FUEL，費用 2 廢料 |
| 隨機事件偶發 | 視事件而定 |

### 設計意圖

FUEL 終於作為策略資源接入機制。「沒油就只能賭」是廢土殘酷感的高峰：不是遊戲逼你死，是你自己把油燒完了。

## Feature 三：敵方追逐 AI

### 設計

敵方不再全程最高速行駛。當 `gap > 敵方武器射程` 時，敵方自動減速追近玩家。

```
if gap > enemy.wRange:
    enemy.spd = min(enemy.maxSpd, player.spd + 追逐加速度)
else:
    enemy.spd = enemy.maxSpd
```

### 設計意圖

解決「速度慢 + 射程短」的僵局：敵方有攻擊動機，不會無限拉開距離。玩家速度若仍不足，結果是「被追上然後挨打」而非「永遠出不了射程、只能無聊撤退」。

玩家的速度優勢依然有意義：速度快可以先進場、先開砲、控制距離。速度劣勢的代價是「敵方能追你，戰場在敵方射程內打」，這是真實的懲罰，不是僵局。

## Feature 四：玩家檔位選擇（回合制速度控制）

### 設計

戰鬥中每隔固定秒數，暫停並讓玩家選擇**下一段的行進檔位**，選完後繼續自動戰鬥。

### 檔位

| 檔位 | 速度    | 效果                        |
|------|---------|-----------------------------|
| 高速 | maxSpd  | 縮短 gap，快速進入/脫離射程 |
| 低速 | maxSpd × 0.5 | 拉開 gap，讓遠程武器發揮  |

（是否需要第三個「中速/維持」檔位，待定）

### 時序

- 每個回合長度：**3 秒**（約一個武器 CD 循環）
- 選擇方式：**暫停選擇**（戰鬥停下，玩家點選，然後繼續）
- 選擇畫面呈現：兩個按鈕 + 當前 gap / 武器狀態資訊

### 設計意圖

讓「射程」成為玩家主動管理的戰術軸：
- 裝了 Mortar 就應該維持低速，壓在敵方射程外
- 裝了 Flamer 就要敢衝高速，進場後才能噴火
- 敵方 AI 的追逐行為讓低速不是免費的——敵方會跟上來

### 待決定

- 2 檔還是 3 檔
- 高速是否消耗 FUEL（讓 FUEL 與戰鬥機制直接連動）
- 首回合是否需要選擇，或第一段預設高速

## 實作優先序建議

| 優先 | Feature | 理由 |
|------|---------|------|
| 1    | 敵方追逐 AI | 解決僵局，影響最大，改動最小 |
| 2    | 節點情報顯示 | UI 層改動，直接提升決策品質 |
| 3    | FUEL 跳過節點 | 需要補充管道一起設計 |
| 4    | 檔位系統 | 最複雜，需要 UI 新增 |
